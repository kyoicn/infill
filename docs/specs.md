# 3D打印作坊生产管理与排班系统 — 详细设计规格

> 本文档是开发的基准文档，所有实现以此为准。如需变更，先更新本文档再修改代码。

---

## 1. 项目概述

一个面向个人3D打印小作坊的生产管理系统，核心功能是管理产品目录、订单、组件库存，并自动生成打印机排班表。

### 业务背景

- 业务内容是生产和销售3D打印的小玩具
- 每个产品由若干3D打印组件组装而成，最简单的产品只有1个组件，最复杂的约5~6个组件
- 一台打印机一次只能打印一种组件，但可以一次打印多个同一组件（如一次打印3个组件A）
- 同一组件的不同打印数量对应不同的打印配置，配置之间是非线性关系（打印2个不一定是1个的2倍时间），且不一定支持所有数量（如组件A可能只支持打1个或3个，不支持2个），这取决于打印版上的物理摆放
- 所有打印配置由用户手动定义和维护
- 目前有4台性能一致的打印机，数量后续可能增加
- 目前有不到10种产品

### 核心需求

- **产品目录管理**：维护产品、组件、组件的打印配置、产品的BOM清单（组装一个产品需要哪些组件各多少个）
- **订单管理**：录入新订单（包含若干产品及数量），查看和管理待处理订单队列，标记订单为已发货
- **组件库存管理**：跟踪各组件的实时库存数量；订单发货时自动扣减库存；支持手动调整库存（应对组装损耗、盘点差异等）
- **打印机排班**：根据待处理订单、产品目录、当前库存，自动生成所有打印机的每日排班表

### 排班约束

- 所有打印任务的启动、收取成品（收菜）、更换打印版均需用户手动操作，无法自动化
- 每次任务结束后需预留15分钟（可配置）用于收菜、换版、启动下一轮
- 用户有固定的操作时间窗口（可按星期几配置，如工作日 08:00-12:00 / 12:30-18:00 / 18:30-23:00，周末不同），窗口外用户不可操作（睡觉、吃饭等）
- 打印任务只能在操作窗口内启动，但可以跨窗口运行（如11:45启动的任务可以在13:45结束，中间午休不影响）
- 当天最后一批任务可以跑过夜，无时长上限，第二天第一个操作窗口开始时收菜
- 多台打印机应尽量在相近的时刻结束任务（约10分钟窗口内），以便用户集中处理，而非反复跑去打印机旁
- 允许子组同步：不要求所有打印机同时结束，如2台先结束并启动新任务、另2台稍后结束，只要每次操作能集中在一个短时间窗口内即可
- 尽量减少打印机空闲时间，最大化利用率

### 富余生产策略

- 实际生产中必然会有富余组件（打印配置不一定恰好匹配需求量），系统需跟踪库存
- 界面显示当前库存折算为多少富余产品的量
- 提供开关让用户决定是否在有剩余产能时继续生产富余组件
- 库存已足够覆盖所有待处理订单加上适量富余时，不再额外生产

### 订单优先级

- 软FIFO：大致按订单创建时间排序，优先生产早期订单所需组件
- 不死板：如果当前库存已能满足某个早期订单，可跳过它，优先生产后续订单缺少的组件
- 库存足够直接满足的订单可以直接发货，不受队列顺序限制

### 每日工作流

- 每天有一个"晚间盘点"环节（时间弹性，用户自选，如20:00~24:00之间均可）
- 盘点内容：录入当天新订单、标记已发货订单、手动调整库存、生成第二天的排班表
- 生成的排班表允许用户手动调整（删除/替换任务、增减批次）后确认
- 第二天按排班表执行：在操作窗口内启动任务、收菜、换版

### 通用要求

- 单用户使用
- 仅支持中文界面
- Web应用，浏览器访问

## 3. 数据模型

### 3.1 组件 (Component)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| name | string | 组件名称 |
| description | string | 描述（可选） |

### 3.2 打印配置 (PrintConfig)

每个组件可以有多种打印配置，由用户手动定义。配置之间的关系是非线性的（打印2个不一定是打印1个的2倍时间，且不一定支持所有数量）。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| component_id | int | 所属组件 |
| quantity | int | 一次打印的数量 |
| duration_minutes | int | 耗时（分钟） |

示例：组件A的打印配置可能是：
- 1个，60分钟
- 3个，150分钟

（不支持一次打印2个——因为打印版摆放限制等原因）

### 3.3 产品 (Product)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| name | string | 产品名称 |
| description | string | 描述（可选） |

### 3.4 产品BOM (ProductComponent)

产品与组件的多对多关系，描述组装一个产品需要哪些组件及数量。

| 字段 | 类型 | 说明 |
|---|---|---|
| product_id | int | 产品ID |
| component_id | int | 组件ID |
| quantity | int | 所需数量 |

### 3.5 订单 (Order)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| created_at | datetime | 创建时间 |
| status | enum | 待处理 / 已发货 |
| shipped_at | datetime | 发货时间（可选） |

### 3.6 订单明细 (OrderItem)

| 字段 | 类型 | 说明 |
|---|---|---|
| order_id | int | 订单ID |
| product_id | int | 产品ID |
| quantity | int | 数量 |

### 3.7 库存 (Inventory)

| 字段 | 类型 | 说明 |
|---|---|---|
| component_id | int | 组件ID（唯一） |
| quantity | int | 当前库存数量 |

### 3.8 打印机 (Printer)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| name | string | 名称/编号（如"1号机"） |

当前4台，后续可增减。所有打印机性能一致。

### 3.9 操作时间窗口配置 (ScheduleConfig)

按星期几配置用户可操作打印机的时间段。

| 字段 | 类型 | 说明 |
|---|---|---|
| day_of_week | int | 0=周一, 6=周日 |
| windows | list | 时间段列表，如 [{"start": "08:00", "end": "12:00"}, ...] |

示例：
- 周一至周五：08:00-12:00, 12:30-18:00, 18:30-23:00
- 周末：09:00-12:00, 13:00-18:00, 19:00-23:00

### 3.10 系统配置 (SystemConfig)

| 配置项 | 默认值 | 说明 |
|---|---|---|
| changeover_minutes | 15 | 收菜+换版+启动下一轮的时间（分钟） |
| surplus_enabled | true | 是否生产富余组件 |

### 3.11 排班表 (PrintPlan)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| date | date | 排班日期 |
| status | enum | 草稿 / 已确认 |
| created_at | datetime | 生成时间 |

### 3.12 排班批次 (PrintBatch)

一个批次代表一组同时启动的打印任务。

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| plan_id | int | 所属排班表 |
| start_time | time | 批次启动时间 |
| batch_order | int | 批次序号 |

### 3.13 排班任务 (PrintTask)

| 字段 | 类型 | 说明 |
|---|---|---|
| id | int | 主键 |
| batch_id | int | 所属批次 |
| printer_id | int | 分配的打印机 |

## 4. 每日工作流

```
晚间盘点（弹性时间，用户自选，如20:00~24:00之间）
  ├─ 输入今天新到的订单
  ├─ 标记今天已发货的订单 → 系统自动扣减对应组件库存
  ├─ 手动微调库存（如组装损耗）
  └─ 生成明天的排班表 → 可手动调整 → 确认

第二天执行排班
  ├─ 第一个操作窗口开始 → 启动第一批打印任务
  ├─ 每批任务结束 → 15分钟收菜换版 → 启动下一批
  ├─ 操作只在操作时间窗口内进行
  └─ 最后一批可跑过夜 → 第二天早上第一个操作窗口收菜
```

---

## 5. 排班算法

### 5.1 第一步：计算组件需求

```
各组件需求量 = 所有待处理订单的组件需求总和 + 富余量 - 当前库存
```

**富余策略：**
- 界面显示当前库存相当于多少个富余产品（按各产品BOM换算）
- 提供开关：是否继续生产富余组件
- 如果开启富余生产，在排班有剩余产能时安排富余打印
- 不过度生产——库存已足够覆盖订单加上适量富余时，不再额外生产

### 5.2 第二步：选择打印配置组合

对每个有需求的组件，从其可用的打印配置中选出一组配置组合，使打印总量 ≥ 需求量，且尽量减少浪费。

例：需要5个组件A，可用配置为 [1个, 3个]，则选 3+3=6（浪费1个）优于 1×5=5（5次打印占用太多时间）。

### 5.3 第三步：分批次调度

**核心规则：**

1. 将所有待打印任务放入候选池
2. 每个批次从池中挑选最多N个任务（N = 打印机数量）
3. **同步策略**：优先选择耗时相近的任务组成一批，使打印机尽量在相近时间完成
4. **允许子组同步**：不要求所有打印机同时结束。例如4台机器中2台先结束可以先启动新一批，另2台稍后结束再启动——只要操作可以集中在一个短时间窗口内（约10分钟）完成即可
5. **操作窗口约束**：批次的启动时间必须落在操作时间窗口内
6. **跨窗口运行**：打印任务可以跨越操作窗口运行（如11:45启动，13:45结束，中间午休不受影响）
7. **过夜任务**：当天最后一批可以运行到第二天，无时长上限，第二天第一个操作窗口开始时收菜
8. **填充产能**：如果某批次中某台打印机没有必须的任务，且富余开关开启，可安排打印富余组件

**订单优先级：**
- 软FIFO：大致按订单创建时间排序，优先生产早期订单所需组件
- 不死板：如果库存已能满足某个早期订单，可跳过它优先生产后续订单缺少的组件
- 库存能直接满足的订单可以直接标记为可发货

## 6. 库存管理

### 6.1 库存增加
- 打印完成后，用户手动确认"完成"，对应组件入库

### 6.2 库存扣减
- 用户标记订单为"已发货"时，系统根据订单明细和产品BOM自动扣减对应组件库存

### 6.3 手动调整
- 用户可以随时手动调整任意组件的库存数量（应对组装损耗、盘点差异等）

---

## 7. 排班手动调整

生成排班表后，用户可以进行以下调整：

- **删除任务**：移除某个批次中某台打印机的任务
- **替换任务**：将某个任务的打印配置换成另一个（如换成打另一种组件）
- **增加批次**：在排班末尾追加一个批次
- **减少批次**：删除整个批次

调整后系统自动重新计算后续批次的时间。

## 8. 界面页面结构

### 8.1 仪表盘（首页）
- 今日排班概览（甘特图）
- 待处理订单数量
- 库存预警（低库存组件提示）
- 富余状态：当前库存相当于多少个富余产品

### 8.2 产品目录
- 产品列表（CRUD）
- 每个产品的BOM清单管理
- 组件列表（CRUD）
- 每个组件的打印配置管理

### 8.3 订单管理
- 新增订单（选择产品和数量）
- 待处理订单列表（按创建时间排序）
- 标记订单为已发货 → 自动扣减库存
- 已发货订单历史

### 8.4 库存管理
- 各组件当前库存数量
- 手动调整库存（增减）
- 库存相对于待处理订单的富余情况展示

### 8.5 排班中心
- 生成排班表（选择日期，一键生成）
- **甘特图视图**：横轴时间，纵轴打印机，任务为色块，操作窗口高亮
- **列表视图**：按批次分组，每批显示启动时间和各打印机的任务详情
- 手动调整排班（删除/替换任务，增减批次）
- 确认排班

### 8.6 系统设置
- 打印机管理（增减打印机）
- 操作时间窗口配置（按星期几设置）
- 换版时间配置
- 富余生产开关
