# 背景
我在运作一个小型3D打印业务，主要生产和销售一些小玩具。这些玩具通常由多个3D打印的组件组成，然后组装在一起。我需要管理生产过程，特别是3D打印机的调度，以优化整体生产效率。

一般一个3D打印机一次只能打印一种组件，但有可能一次性打印多个同一组件，比如一次性打印3个组件A或者10个组件B。对于同一个组件，也可能有不同的配置，比如打印3个组件A，或5个组件A。打印不同组件需要的时间不同（主要取决于个数）。我会编写一个产品目录，列出所有的产品以及组装它所需要的组件个数，之后我也会持续往这个目录中添加新的产品和组件打印配置。举个例子：
产品A：组件A1 x 2, 组件A2 x 1
可能的打印配置：
- S1: 1个组件A1，耗时1小时
- S2: 2个组件A1，耗时2小时
- S3: 3个组件A1，耗时3小时
- S4: 1个组件A2，耗时1小时
- S5: 2个组件A2，耗时2小时
- S6: 3个组件A2，耗时3小时

同时我每天会有新的订单进来，里面会写明需要多少个产品A，多少个产品B等等。比如：
订单1：产品A x 2
订单2：产品A x 1, 产品B x 1

这些新进来的订单会加入待处理订单队列（早进来的订单应该早生产）。然后每天发走的订单应该从这个队列中移除。

我们不可能永远只生产需要用到的组件，我们肯定会生产更多的富余组件，以备未来的订单使用，所以也许要记录当前各个组件的库存。

我目前有四台3D打印机，他们的性能是一致的，之后可能会增加打印机的数量。

# 需求

我希望边写一个软件来帮助我给打印机排班。软件需要有以下功能：
* 有一个友好方便的界面，我可以配置产品目录，打印机数量，输入新的订单，查看和管理待处理订单，查看管理组件库存等等。
* 软件需要能够根据订单和产品目录，自动生成针对所有打印机的排班表。排班表需要满足以下条件：
    * 以24小时为周期，每天的0点开始
    * 要考虑到每天0点到8点为我的休息时间，这期间打印机可以工作，但我没法收菜以及更换打印任务
    * 每次打印任务结束之后，都需要预留一些时间给我去收菜并更换打印版
    * 所有打印机应该尽量在相近的时刻结束工作，这样我可以一次性收菜并更换打印版，不需要时不时跑去打印机那边
    * 尽可能的减少打印机的空闲时间，最大化利用率
    * 可以多打印一些组件，以备未来订单使用
* 只需要支持中文

总之，这是给我的个人小作坊使用，和考虑到我的时间安排，比如你可以安排一个打印任务在凌晨3点结束，但也要知道我早上8点才起床收菜。
